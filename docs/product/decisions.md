# Decisões

## Finance Core antes do ecossistema completo

Motivo: finanças são o chão do produto. Marketing e tracking dependem de uma base confiável de dados, documentação e deploy.

## Pessoal e empresarial separados

Motivo: evita confusão entre vida pessoal e operação. Uma visão consolidada poderá existir depois, se fizer sentido.

## Caixa prático

Motivo: entrega clareza rápida sobre entradas, saídas e saldo sem exigir contabilidade completa.

## Next.js, Supabase e Vercel

Motivo: combinação simples para app web, autenticação, banco de dados e publicação rápida.

## Docs vivos no repositório

Motivo: Codex e Claude Code precisam entender o estado do projeto sem depender de memória de chat.
